Extract

command line tool

  • Used on: Panama Papers (19M docs), Swiss Leaks (116K docs) and Luxembourg Leaks (1K docs)
  • LOC (Java): 10,961
  • Website: github.com/ICIJ/extract

A versatile framework that allows to scan a filesystem, enqueue paths, and extract (with Tika/tesseract) content into a document tree (for each document), then allow to "spew" extracted text into SolR, files or standard output. It has been used widely on several ICIJ stories.

Datashare

a proof of concept that was doing extract features + NLP analysis

  • Never used in "production"
  • LOC (Java): 9,854
  • Website: github.com/ICIJ/datashare

It was a whole workflow from filesystem scanning to NLP pipelines. It was multithreaded with a near actor model (blocking & forwarding queues) for all steps (scanning, parsing, indexing, NLP pipelines) extensible to a processing distributed system based on Hazlecast. It was using Elasticsearch that has great clustering features. It was implementing 5 NLP pipelines (CoreNLP, OpenNLP, IxaPipe, Mitie, GateNLP). It was packaged to be used on a standalone computer

Knowledge Center (aka KC)

An updated version of Blacklight that is a GUI on top of SolR

  • Used on:Panama Papers (19M docs), paradise paper (10M docs), Swiss Leaks (116K docs), Luxembourg Leaks (1K docs)
  • LOC (Ruby): A lot
  • Website: projectblacklight.org

"Knowledge Center" is an updated version of Blacklight. It is a ruby on rails framework on top of SolR index to build websites that needs to search on indexed data. It is used by libraries, universities, registers, etc. It has a lot of indexed data (around 35 Millions of documents), and was used by around 300 ICIJ journalists and partners on several projects.
